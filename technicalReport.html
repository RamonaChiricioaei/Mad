1. Introduction

MAd-Medical Web Advisor is a hypermedia platform developed with the scope to be an elearning platform for middle and high school students.

The main idea is that not everyone has the skills to access knowledge sources like DBpedia or Wikidata and in this case we are trying to develop a more friendly application that will display a list of medical conditions-such as common diseases, allergies, food intolerances, obesity factors, disorders etc- and how this conditions can affect the human body, likewise the user can see how a particular medical condition is influenced or not by geographical and / or cultural factors.

The site will provide an important method of searching medical conditions only by introducing the symptoms, also will provide recommendations about an optimal lifestyle-including both mind and body-and the user can see a dataset with approved drugs and their therapeutic indications.

----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

2. Architecture

The arhitecture is based on MVC design pattern.

M-model (the way data is represented- ex: medicalEntityDTO, searchDTO- classes containing attributes)

V- view (the way the data is displayed in GUI)

C- controller (all methods used to manipulate the data )

The main modules of the application are: Medical conditions, Statistics, Search based on symptoms and Healthy Lifestyle.

MAd-Medical Web Advisor is a client-server application that use external data repository. The main data is retrieved from DBpedia and Wikidata . An SPARQL Endpoint is used to retrieve data. The client is implemented using JSF tehnology, javascript and ExpoFacts (to display the results). On server, there is implemented a REST API that expose more methods used by client to obtain the information desired.

3. Frond-end

Start page: contains all the important things that you can do in this application: Medical condition, Statistics, Search based on symptoms and Healthy lifestyle.

Medical condition Page: contains 5 specific medical conditions: common diseases, allergies, food intolerances, obesity factors, disorders. In each of these pages you' ll find a sorted list of links of common diseases/allergies/food intolerances/obesity factors/disorders; by clicking on a link a new page we'll be displayed; this page contains 4 more important things about the medical condition: Description, Cause, Symptoms, Treatament and More information(nothing special here!).


Statistics Page: here you can find how a medical condition can affect the human body based on specific conditions(geographical and cultural factors): here we will use ExpoFacts and chart to display the dataset.



Search based on symptoms Page: based on symptoms you selected the application we'll display medical conditions that have those symptoms.

Healthy lifestyle Page: with three elements : body, mind and approved drugs. Body and mind represents suggestion for a healthy lifestyle and approved drugs is a dataset with drugs and their therapeutic indications.

5. Conclusions

MAd-Medical Web Advisor will be a site that provides useful informations from the medical branch for unknowing users and also for more experienced users.

Medical conditions and how geographical/cultural factors can affect the human body are trivial things in this domain(because there are many specialized sites in this area), but the non-trivial services are Searching based on symptoms and Approved drugs and their therapeutic indications.

Service based architecture makes that this platform to be easily extended to something more important and more useful in medical domain.
